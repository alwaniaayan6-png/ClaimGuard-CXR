# ClaimGuard-CXR — Reproducibility Guide

**Paper:** "Calibrated Claim-Level Hallucination Detection for Radiology Report Generation with Conformal FDR Control"
**Target venue:** NeurIPS 2026 (May 6 deadline)
**Last updated:** 2026-04-14

This document describes how to reproduce every result in the paper from the
CheXpert Plus dataset. All experiments run on Modal (cloud GPU) except
baselines which run locally.

## Scope and trust model (important)

ClaimGuard-CXR is a **text-based** verifier. It scores textual consistency
between a claim and its supplied evidence using a binary RoBERTa-large
cross-encoder. It does NOT read pixel-level image data at inference time,
and it does NOT verify provenance on its own — it trusts the evidence it
is handed.

To prevent self-consistency loops (the same model generating both the
claim and the "evidence"), ClaimGuard-CXR ships with a pipeline-level
**provenance gate** that only certifies claims whose evidence comes from a
trusted or independent source. The verifier still scores every claim, but
the final triage label depends on `evidence_trust_tier`:

| `evidence_trust_tier` | Source example                                     | Can be certified safe? |
|-----------------------|-----------------------------------------------------|------------------------|
| `trusted`             | Oracle text from the same radiologist report        | Yes (`supported_trusted`) |
| `independent`         | MedCPT-retrieved human-written passage; cross-model generator output | Yes (`supported_trusted`) |
| `same_model`          | Evidence generated by the same LLM/VLM as the claim | **No** (`supported_uncertified`) |
| `unknown`             | Legacy data with no provenance annotation           | **No** (`supported_uncertified`) |

Every claim-evidence record written by `scripts/prepare_eval_data.py` and
`scripts/modal_build_retrieval_eval.py` now carries these five provenance
fields in addition to the existing ones:

```json
{
  "evidence_source_type": "oracle_report_text" | "retrieved_report_text" | "generator_output" | "unknown",
  "evidence_is_independent": true | false,
  "evidence_generator_id": null | "CheXagent-8b" | ...,
  "claim_generator_id": null | "CheXagent-8b" | ...,
  "evidence_trust_tier": "trusted" | "independent" | "same_model" | "unknown"
}
```

Legacy eval files that predate this patch can still be loaded; they are
backfilled to `evidence_trust_tier = "unknown"` at load time and therefore
cannot be certified safe. See `inference/provenance.py` for the canonical
constants and `tests/test_provenance.py` for the complete truth table.

## Key hyperparameters (verifier training)

| Parameter | Value |
|---|---|
| Base model | `roberta-large` |
| Batch size | 32 (× 2 gradient accumulation = effective 64) |
| Epochs | 3 |
| Learning rate | 2e-5 (linear decay, 10% warmup) |
| Weight decay | 0.01 |
| Label smoothing | **0.0** (critical — 0.05 creates conformal score ceiling) |
| Dropout | 0.1 |
| Max seq length | 512 |
| Optimizer | AdamW |
| Val split | 10% patients (stratified) |
| Best ckpt | lowest val_loss |
| Early stopping | patience=1 on val_loss |
| Determinism | cudnn.deterministic=True + use_deterministic_algorithms=True |
| Seed | 42 |

## Key conformal hyperparameters

| Parameter | Value |
|---|---|
| Procedure | Inverted label-conditional cfBH (Jin & Candes 2023, adapted) |
| Calibration pool | Contradicted test claims (label==1), one per patient (C7) |
| Score | P(not-contradicted) from temperature-calibrated softmax |
| p-value | `(|cal_contra_score >= test_score| + 1) / (n_cal + 1)` |
| BH scope | **Global** (not per-pathology) — per-group BH is too strict |
| α levels | {0.01, 0.05, 0.10, 0.15, 0.20} |
| Temperature | LBFGS on calibration NLL |

## Data generation contract

- **Patient separation:** no patient in more than one split
- **Hard-neg types** (8, equal frequency): negation, laterality_swap,
  severity_swap, temporal_error, finding_substitution, region_swap,
  device_line_error, omission_as_support
- **Contradiction validator:** perturbed claim verified to NOT already be
  supported by original evidence (avoids label noise bug C6)
- **Insufficient examples:** evidence from DIFFERENT patient AND DIFFERENT
  pathology (avoids leakage bug C5)
- **Laterality safety (M8):** only swap laterality in claims that aren't
  bilateral in the original report

## Known caveats

1. **Label smoothing must be 0** for binary training. Setting to 0.05
   creates a hard ceiling at P=0.975 that collapses the conformal score
   distribution — BH accepts nothing (n_green=0 at all α). The 3-class
   model used label smoothing 0.05 and had the same issue.

2. **Global BH (not per-group)** is required for the conformal procedure.
   Per-group BH fails because each pathology's cal pool has min_p > local
   BH threshold. We compute per-pathology p-values for exchangeability,
   then apply BH globally.

3. **Inverted conformal** — calibrate on CONTRADICTED (null H0=contra),
   not on faithful. Calibrating on faithful fails because both cal and
   test faithful scores sit at the softmax ceiling.

4. **BM25 sparse retrieval is built but not used at eval time** — rank_bm25
   per-query scoring is O(corpus) (~2s for 1.2M passages), infeasible at
   30K eval claims. Dense-only MedCPT retrieval gives 98.09% accuracy
   with only 0.22pp drop from oracle. Batched sparse retrieval is future
   work.

5. **Text-only verifier; provenance gate required for trust certification.**
   The verifier does not look at images. It scores textual consistency
   between a claim and supplied evidence. If the same LLM generated both
   the claim and the "evidence", the verifier's high score is meaningless.
   The pipeline-level provenance gate (`inference/provenance.py`) refuses
   to label such a claim `supported_trusted` regardless of the verifier's
   score or the conformal BH decision. The reported FDR guarantees in the
   paper hold only for the `trusted` and `independent` trust tiers.
   Same-model generated evidence is supported by the data contract and
   gated by policy, but is not present in the CheXpert Plus benchmark
   results — it is scaffolded for future work on real LLM-generated
   radiology outputs.

6. **Legacy eval data is auto-backfilled to `unknown`.** Records written
   before this patch lack the five provenance fields. They are not
   rejected at load time; they are backfilled to
   `evidence_trust_tier = "unknown"`, which the gate treats as
   non-certifiable. This means re-running old eval data under the new
   pipeline produces the same verifier scores but a conservative "no
   supported_trusted labels" output. Regenerate with
   `scripts/prepare_eval_data.py` to get full provenance stamps.
